# AGENTS.md — Quick guide for AI coding agents

Purpose: provide brief, actionable instructions so an AI coding agent can become productive quickly in this repository.

## Quick commands
- Install: `npm install`
- Dev (express + vite via Portless): `npm run dev` (see `scripts/dev.mjs`)
- Build frontend: `npm --prefix frontend run build`
- Type-check frontend: `npx tsc -p frontend/tsconfig.json --noEmit`
- Run backend tests: `npm test` or `npx vitest run backend/src/routes/locations.test.ts`
- Reset DB: `npm run reset`

## What this repo is
- Fullstack TypeScript starter: Express backend + Vite + React frontend.
- Backend persists location snapshots in SQLite via Drizzle ORM.
- Frontend is React + Tailwind and uses a local `StoreProvider` for app state.

## Conventions & tips for agents
- Keep backend routes and DB schema stable unless the user asks for schema changes. Migrations live in `backend/drizzle` — run `npm run db:migrate` after altering `schema.ts`.
- Frontend and backend are developed together in dev via Portless. Use the Portless host printed by `npm run dev` (e.g., `http://weather-starter.localhost:1355`).
- Tests are primarily backend API tests (Vitest + Supertest). Run targeted tests when modifying routes or DB code.
- Format and linting: project uses `tsc` and Tailwind; follow existing Tailwind utility classes.
- When adding UI features, prefer adding new components under `frontend/src/components` and keep styling in `index.css` or the component file.

## Quick debugging notes (useful historical/issues)
- Portless on Windows may register a hostname and prevent re-run; use `taskkill` to stop lingering Node processes if `scripts/dev.mjs` reports an already-registered name.
- The app uses `data.gov.sg` realtime endpoints. When adding new readings, reuse `fetchNearestReading()` pattern in `backend/src/weather.ts`.
